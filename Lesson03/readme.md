# Lesson03 - 静态渲染与 Surface 画布基础

## 学习目标

1. 理解 Pygame 中 `Surface` 作为画布的核心概念；
2. 掌握基础几何绘制、图片贴图 `blit`、图层顺序；
3. 理解颜色、透明通道、z-order 层级。
4. 为什么需要虚拟分辨率（跨分辨率适配、坐标系统一）

## 重点理解

- 游戏画面本质：在 `Surface` 上执行绘制操作，通过 `blit` 完成贴图叠加。
- 渲染顺序：后绘制的覆盖先绘制的，对应 Godot 中 `z\_index` 或场景树顺序。
- 透明通道：`convert\_alpha\(\)` 保留 PNG 透明度。
- 虚拟分辨率与视口映射：为什么需要虚拟分辨率（跨分辨率适配、坐标系统一）；固定虚拟 Surface → 计算缩放比例与黑边偏移 → 最终 blit 到窗口。

## 动手练习题

1. 绘制矩形、圆形、线、多边形。
2. 加载精灵图片，`blit` 显示。
3. 绘制 3 个重叠矩形，调整顺序观察遮挡效果。
4. 实现一个可缩放窗口，按比例显示一个移动的方块，支持切换“黑边/拉伸”模式。设定逻辑分辨率（如 800×600），绘制到中间 Surface，最后缩放至实际窗口。实现保持宽高比的 letterbox 模式（上下或左右黑边）。扩展：鼠标坐标映射（虚拟 → 物理）。

## 🔁 映射引擎原理

- 在 Godot 中创建两个 `Sprite2D`，调整 `z\_index`，观察遮挡。对照 Pygame 的 blit 顺序。
- 类比 Godot 的 `stretch` 模式与 `Viewport` 的 `size。` 设置 `Viewport` 的拉伸模式，对比你的实现。
