# Lesson15 - 分层架构与模块设计规范

## 学习目标

1. 掌握「数据层 - 逻辑层 - 表现层」三层分离；
2. 理解依赖倒置原则（高层不依赖低层）；
3. 规范大型项目的目录结构。

## 重点理解

- 三层职责：
  - 数据层：配置只读数据、存档读写。
  - 逻辑层：状态机、物理、经济规则等。
  - 表现层：渲染、UI、输入反馈。
- 依赖规则：表现层可以依赖逻辑层，逻辑层不能直接依赖表现层（通过事件通信）。
- 目录示例：

project/

data/ (configs, saves)

logic/ (fsm, physics, systems)

presentation/ (renderer, ui, input\_handler)

common/ (event\_bus, resource\_manager)

## 动手练习题

1. 将第二阶段综合案例按三层架构重构代码。
2. 消除所有全局变量，使用依赖注入或事件通信。
3. 为每个模块编写简洁的接口文档（docstring）。

### 🔁 映射引擎原理

- 在 Godot 项目中，尝试将自定义脚本分离为“数据资源（Resource）”和“逻辑控制器”。
- 思考：Godot 的节点树天然混合了逻辑与表现，你认为这种混合的优点和缺点是什么？能否通过某种模式实现类似三层架构？
