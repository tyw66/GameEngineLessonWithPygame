# Lesson14 - 事件系统与模块解耦通信（发布-订阅模式）

## 学习目标

1. 解决模块紧耦合问题；
2. 手写事件总线（注册/发布/注销）；
3. 对应 Godot 信号机制底层原理。

## 重点理解

- 紧耦合痛点：A 直接调用 B 方法，导致修改 B 时必须修改 A。
- 事件总线：发布者 `emit\(event\_name, data\)`，订阅者 `on\(event\_name, callback\)`，两者无直接依赖。
- 引擎对照：Godot 中的 `signal` 本质上是对象间的事件机制，但更绑定于特定对象；事件总线是全局的。

## 动手练习题

1. 实现 `EventBus` 类（字典存储事件名 → 回调列表）。
2. 将第二阶段案例中的“拾取金币”、“玩家死亡”等改为事件驱动。
3. 扩展支持事件优先级和一次性订阅。
