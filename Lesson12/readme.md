# Lesson12 - 粒子系统与批量特效底层原理

## 学习目标

1. 理解粒子系统本质：批量生成短生命周期 + 运动 + 渐变；
2. 手写粒子类、发射器，实现火焰、爆炸、雪花特效；
3. 了解 Pygame CPU 粒子的性能上限（一般 \&lt; 2000 粒子）。

## 重点理解

- 粒子四要素：生成器（位置/速率）、初始属性（速度/颜色/大小/寿命）、运动逻辑（重力/阻尼）、消亡回收。
- 批量管理：将所有粒子存在一个列表中，每帧更新并绘制，移除死亡粒子。
- 性能瓶颈：纯 Python 循环 + 绘制每个粒子会明显降低帧率。现代引擎使用 GPU 或 C++ 后端。
- 为什么 GPU 更快：粒子属性可存储在 GPU 数组中，并行计算位置和生命周期。

## 动手练习题

1. 实现 `Particle` 类（位置、速度、颜色、大小、寿命、衰减速度），每帧更新并淡出。
2. 实现 `ParticleEmitter`：控制发射频率（每帧累积时间，到达间隔则发射新粒子），随机速度和颜色。
3. 做出三种特效：火焰（向上发射，橙红）、爆炸（向外散射，快速消失）、雪花（飘落，轻微左右摆）。
4. 性能测试：发射 3000 粒子，记录帧率，思考优化方案（减少每粒子独立绘制、使用 subsurface 纹理）。

- 思考：你的手写粒子系统与 `GPUParticles2D` 在实现机制上有何本质不同？你能否模拟它的“颜色渐变”功能？
