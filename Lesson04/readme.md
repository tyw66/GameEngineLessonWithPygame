# Lesson04 - 输入系统与交互判定

## 学习目标

1. 掌握键盘/鼠标的两种检测模式：事件触发（单次） vs 状态轮询（长按）；
2. 手写实现点击移动、长按移动、UI、hover、拖拽；
3. 理解坐标转换、点-矩形/点-圆碰撞判定。

## 重点理解

- 事件触发：`KEYDOWN`、`MOUSEBUTTONDOWN`，适合单次操作，如菜单选择、跳跃等。
- 状态轮询：`pygame.key.get_pressed()`、`pygame.mouse.get_pressed()`，适合持续操作，如移动、拖拽等。
- UI交互本质：鼠标坐标 + 几何碰撞检测。

## 动手练习题

1. 鼠标坐标跟踪、左右键点击、方向键长按移动方块。
2. 手写按钮：默认/hover/点击三种状态，带回调。
3. 可拖拽矩形：鼠标按下时选中，移动时更新位置。

## 🔁 映射引擎原理

- 在 Godot 中用 `_input(event)` 处理点击，用 `Input.is_action_pressed()` 处理长按。对比两种方式的适用场景。
