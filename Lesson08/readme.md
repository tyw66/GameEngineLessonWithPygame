# Lesson08 - 资源管理与静态配置系统

## 学习目标

1. 理解资源加载、缓存、释放的工程逻辑；
2. 实现简易资源管理器，避免重复 IO；
3. 掌握数据驱动开发：配置与逻辑分离。

## 重点理解

- 懒加载：首次使用时加载，存入缓存池。
- 缓存策略：字典映射路径 → 资源，再次请求直接返回。
- 配置分离：JSON/YAML 存放数值、道具属性、关卡数据，代码读取并驱动逻辑。

## 动手练习题

1. 实现 `ResourceManager`，支持图片、字体、音效的懒加载与缓存。
2. 读取 JSON 物品配置表，在界面中列出所有物品及其属性。
3. 修改 JSON 文件中的数值（如攻击力 10→20），不重启程序观察数据变化（需热重载机制，简单实现为每几秒重新读取）。
