# Lesson01：游戏主循环（最最核心，一切的起点）
 
**目标**

彻底搞懂「事件循环 / 更新循环 / 渲染循环」三者关系

**亲手写一遍**

1. 窗口创建、时钟控制、帧率限制
2. 事件处理（QUIT、KEYDOWN、MOUSEMOTION）
3. 每一帧：清屏 → 更新逻辑 → 绘制所有元素 → 刷新屏幕

**重点理解**

1. 为什么要清屏？（双缓冲、画面残影原理）
2. 输入事件「轮询」和「事件队列」的区别
3. deltaTime 时间步、固定时间步 vs 可变帧率
4. 为什么现代引擎都用这套循环（Godot 的 _process / _physics_process 本质就是它）


**练法**

写一个最小 Demo：窗口 + 移动方块 + 按键控制 + 帧率显示，手写完整循环，不封装任何高级类。
