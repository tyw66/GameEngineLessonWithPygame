# 综合案例 02 —— 极简横版闯关小游戏

## 学习目标

- 综合运用以下所有系统：状态机、资源管理、存档、物理碰撞、动画、粒子、场景图（摄像机跟随）、音频。
- 实现一个可玩的小型游戏，强化多系统协同能力。
- 反向映射到 Godot 的完整工作流。

## 重点理解

- 时间系统驱动所有运动与动画；固定时间步长用于物理（可选，但建议实现简单版本）。
- 摄像机跟随本质是场景树中的坐标偏移。
- 数据与配置分离：角色属性、关卡地形来自 JSON，存档保存玩家进度。

## 动手练习题（可增量实现）

基础要求（必须）：

1. 角色 FSM（站立、行走、跳跃、受伤），动画同步状态。
2. 手写重力 + AABB 碰撞响应（地面、平台）。
3. 场景图：所有游戏物体继承自 `TransformNode`，摄像机跟随玩家。
4. 资源管理器统一加载图片、音效。
5. 音频：跳跃音效、收集音效、背景音乐。
6. 粒子特效：受伤时生成血雾粒子，收集金币时生成星光粒子。
7. 存档系统：保存当前关卡重生点、收集的金币总数。

进阶选项：

- 敌人 AI（简单巡逻或跳跃）。
- 多关卡切换（读新 JSON）。
- UI 血条、分数显示。

## 🔁 映射引擎原理

- 完成游戏后，用 Godot 实现一个更简化的版本（例如 10 分钟原型）。
- 复盘：哪些部分在 Godot 中几乎不需编码（如摄像机跟随、碰撞响应）？哪些部分仍需要编程思维？写出你的对比总结。
