# Lesson16 - 性能优化基础（含空间划分、纹理图集）

## 学习目标

1. 定位 Pygame 常见性能瓶颈；
2. 掌握缓存、空间划分、纹理图集、脏矩形等优化手段；
3. 建立“先实现后优化”的工程原则。

## 重点理解

- Pygame 瓶颈：大量独立绘制调用、频繁 IO、O(N²) 碰撞检测。
- 空间划分：将世界划分为网格，只检测同一或相邻网格内的物体。例：200 个物体，全检测 4 万次，网格划分后可能仅数千次。
- 纹理图集：将多个小图合并为一张大图，用 `Surface\.subsurface` 切出所需区域，减少纹理切换（虽然 Pygame 是 CPU 渲染，但仍可减少 Surface 对象数量）。
- 其他优化：使用 `convert\(\)` 或 `convert\_alpha\(\)` 加速 blit；避免每帧创建新 Surface；限制粒子数量。

## 动手练习题

1. 资源管理器缓存对比：重复加载图片 vs 一次缓存的帧率差异。
2. 实现简单网格空间索引（例如 64x64 网格），用于碰撞优化，测量 200 个随机运动物体的帧率提升。
3. 纹理图集：将 10 张小图标合并成一张大图，并通过 `subsurface` 分别绘制，测量 blit 性能差异（可选）。
4. （高级）脏矩形更新：仅更新鼠标移动影响的小区域，其它区域使用上次绘制结果。
