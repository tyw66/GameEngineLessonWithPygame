# 综合案例 03 —— 分层架构版模拟经营 Demo（带性能优化）

## 学习目标

- 综合运用事件总线、三层架构、性能优化，构建工业化项目架构；
- 反向映射到 Godot 大型项目最佳实践。

## 动手练习题

实现一个模拟经营 Demo（例如简单农场）：

1. 三层严格分离：
   - 数据层：作物配置 JSON、玩家资源（金币、种子数量）存档。
   - 逻辑层：播种、生长计时（基于 delta）、收获事件、经济系统。
   - 表现层：渲染农田网格、UI 按钮、鼠标交互。
2. 模块间通信：全部通过事件总线，如 `EventBus\.emit\(\&\#34;plant\_grown\&\#34;, grid\_x, grid\_y\)`。
3. 性能优化：
   - 使用纹理图集表示作物不同生长阶段。
   - 使用空间划分优化鼠标点选田块时的碰撞检测（对于大网格可选）。
   - 资源全局缓存。
4. 存档：保存每个田块作物的种类和剩余生长时间。
