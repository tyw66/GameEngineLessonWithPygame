# Lesson09 - 存档系统与数据持久化

## 学习目标

1. 掌握玩家动态数据序列化/反序列化；
2. 实现存档、读档、区分静态配置与玩家存档；
3. 基础防篡改（编码/简单加密）。

## 重点理解

- 两类数据：静态配置（只读） vs 玩家存档（读写）。
- 序列化：内存对象 ↔ 文本/二进制数据。
- 简单加密：XOR、base64、或使用 `hashlib` 生成校验值。

## 动手练习题

1. 使用 JSON 保存玩家金币、等级、当前场景坐标，并实现读档。
2. 进阶：将存档内容用 base64 或 XOR 混淆。
3. 对比二进制存档（`pickle`）与 JSON 的读写速度和可读性。

## 🔁 映射引擎原理

- 在 Godot 中使用 `ResourceSaver\.save\(\)` 和 `ResourceLoader\.load\(\)` 保存玩家数据。
- 思考：Godot 的资源系统是如何自动处理序列化循环引用的？
